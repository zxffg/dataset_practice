# Анализ датасетов
Здесь хранится моя проделанная работа связанная с датасетами. Удаление, группировки и практика в работе с данными.

*Основной стек технологий:* Python 3.14, Pandas 3.0.3, Jupyter (VS Code), uv 0.1.0

*Запуск:*
1. Клонирование репозитория.
2. Установка зависимостей и библиотек. Используйте команду `uv sync`
3. Для проссмотра файлов используейте `Jupyter Notebook`
---

#### bank_deposit
Проект: 
Анализ клиентов банка

Цель: 
Проверить факторы, влияющие на открытие депозита.

Датасет: 
[Bank Marketing Dataset](https://www.kaggle.com/datasets/prakharrathi25/banking-dataset-marketing-targets)

Основные выводы:
1. Клиенты с высшим образованием имеют более высокий медианный баланс по сравнению с другими группами.
2. Наличие кредитных обязательств может влиять на склонность клиента к открытию вклада.

Используемые технологии:
Python 3.14, Pandas 3.0.3, Jupyter (VS Code), uv 0.1.0

#### bank_transactions
Проект: 
Анализ клиентов банка, их операций и возможных мошеннических схем.

Цель: 
Проверить связь между подозрительными операциями. Представить модель поведения юзера.

Датасет: 
[Bank Transactions Dataset](https://www.kaggle.com/datasets/valakhorasani/bank-transaction-dataset-for-fraud-detection)

Основные выводы:
1. Высокое количество попыток входа в аккаунт перед транзакцией напрямую связано с увеличенной вероятностью несанкционированного дотсупа (брутфорс)
2. Использование нового DeviceID, который ранее не был привязан к учетной записи, может представлять повышенный фрод-риск. Особенно если сумма операции превышает средний чек пользователя.
3. В определенные часы онлайн-канал испытывает пиковые нагрузки, что приводит к увеличению продолжительности транзакции.

Используемые технологии:
Python 3.14, Pandas 3.0.3, Matplotlib XXX, Jupyter (VS Code), uv 0.1.0
